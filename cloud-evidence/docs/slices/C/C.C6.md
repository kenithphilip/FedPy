## Why this slice exists
NIST SP 800-53 Rev. 5 control **CA-7 (Continuous Monitoring)** mandates a continuous-monitoring program covering organization-defined metrics, frequencies, ongoing assessments, and reporting. **CA-7(1) (Independent Assessment)** requires periodic 3PAO assessment. **PM-31 (Continuous Monitoring Strategy)** is the program-level umbrella. FedRAMP requires a written ConMon Strategy + Plan describing WHICH controls are under continuous monitoring, at WHAT frequency, with WHAT escalation, and to WHICH reporting endpoint. The Strategy is the umbrella; the Plan is the executable cadence. LOOP-E (ConMon agent) reads this document as its configuration.

## Known risks / issues
- **Risk 1 — ConMon Playbook revision.** FedRAMP may publish a v1.1 with different cadence language. Mitigation: module header records retrieval date; emit warning if file age >12 months; operator confirms.
- **Risk 2 — KSI map grep brittleness.** Same as C.C1. Mitigation: assert `ksi_count >= 20` in emitter; structured warning.
- **Risk 3 — Reporting endpoint policy change.** USDA Connect.gov mandate may shift back to per-agency endpoints. Mitigation: §9 sources from R2 finding doc which lists the date of the policy change.
- **Risk 4 — Collaborative ConMon scope ambiguity.** RFC-0026 introduces the term but precise scope is evolving. Mitigation: §12 cites RFC-0026 verbatim + states operator interpretation as REQUIRES-OPERATOR-INPUT-VERIFY.
- **Risk 5 — VDR evidence may miss scanners.** If only AWS Inspector evidence exists but the CSP also runs Snyk + GHAS, §5 under-reports. Mitigation: emit a "discovered scanners" count + a note that operator should add any missing scanners via config.
- **Risk 6 — Escalation thresholds vs CISA BOD vs internal SLA conflicts.** Defaults to CISA BOD 22-01 dueDate semantics but CSP may run tighter; operator overrides via config.

## Open questions (for implementation session to resolve)
- **Q1**: Should §4 mark KSIs that are 100% automated separately from those that require operator action? The `automated_y_n` column is implied but not currently in `ksi-map.ts` shape.
- **Q2**: Should §11 sort escalation thresholds by trigger severity (e.g. KEV first) or by SLA-days ascending? Severity-first is more readable; SLA-days ascending is more actionable.
- **Q3**: How should the document reconcile FedRAMP Rev5 SLA dates with CISA KEV `dueDate`? Both apply; the stricter wins.
- **Q4**: Should §5 emit a per-scanner row even when the scanner has 0% coverage (degenerate)? Or omit?
- **Q5**: Should §13 Plan Maintenance enumerate a SHA256 of the SSP it references? Strengthens chain-of-custody.
